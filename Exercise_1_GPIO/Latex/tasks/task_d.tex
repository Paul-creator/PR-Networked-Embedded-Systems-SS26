\section{Task D: Vary the Blinking Frequency with Push Buttons}

\subsection{a) Implementation:}
Für diese Aufgabe mussten die beiden Taster (PJ12 und PJ13) in STM32CubeMX zunächst von GPIO Output auf \textbf{Input with Pull-Up} umgestellt und der Code anschließend neu generiert werden. Erst danach konnten die Tasterzustände mit \texttt{HAL\_GPIO\_ReadPin()} korrekt eingelesen werden.

Auf dem Board sind die beiden Taster als \texttt{PB3} und \texttt{PB4} beschriftet. Im Programm werden sie über die Signale \texttt{BTN1} und \texttt{BTN2} verwendet, wobei in CubeMX beispielsweise \texttt{PJ13 = BTN2} konfiguriert wurde.

Da Pull-Up verwendet wird, gilt:
\[
	\texttt{GPIO\_PIN\_SET} = \text{Taster nicht gedrückt}, \qquad
	\texttt{GPIO\_PIN\_RESET} = \text{Taster gedrückt}.
\]

Die Zykluszeit wird in Schritten von \(100\,\mathrm{ms}\) zwischen \(100\,\mathrm{ms}\) und \(1000\,\mathrm{ms}\) verändert. BTN1 verkleinert die Zykluszeit, BTN2 vergrößert sie. Die LED wird mit konstantem Duty Cycle von \(50\%\) betrieben, daher gilt:
\[
	t_{on} = \frac{t_{Cycle}}{2}, \qquad t_{off} = t_{Cycle} - t_{on}.
\]

Die Tastererkennung erfolgt über den Vergleich von altem und aktuellem Zustand. Im finalen Code wird auf die Flanke von \texttt{RESET} nach \texttt{SET} reagiert, also auf das \textbf{Loslassen} des Tasters.

\begin{lstlisting}
/* USER CODE BEGIN PV */
uint32_t cycleTime = 500;
const uint32_t minCycleTime = 100;
const uint32_t maxCycleTime = 1000;
const uint32_t stepSize = 100;

GPIO_PinState lastBtn1State;
GPIO_PinState lastBtn2State;
/* USER CODE END PV */

/* USER CODE BEGIN 2 */
lastBtn1State = HAL_GPIO_ReadPin(GPIOJ, BTN1_Pin);
lastBtn2State = HAL_GPIO_ReadPin(GPIOJ, BTN2_Pin);
/* USER CODE END 2 */

/* USER CODE BEGIN WHILE */
while (1)
{
  GPIO_PinState currentBtn1State = HAL_GPIO_ReadPin(GPIOJ, BTN1_Pin);
  GPIO_PinState currentBtn2State = HAL_GPIO_ReadPin(GPIOJ, BTN2_Pin);
  uint32_t onTime = cycleTime / 2;
  uint32_t offTime = cycleTime - onTime;

  if ((lastBtn1State == GPIO_PIN_RESET) && (currentBtn1State == GPIO_PIN_SET))
  {
    if (cycleTime > minCycleTime)
    {
      cycleTime -= stepSize;
    }
  }

  if ((lastBtn2State == GPIO_PIN_RESET) && (currentBtn2State == GPIO_PIN_SET))
  {
    if (cycleTime < maxCycleTime)
    {
      cycleTime += stepSize;
    }
  }

  lastBtn1State = currentBtn1State;
  lastBtn2State = currentBtn2State;
  /* USER CODE END WHILE */

  /* USER CODE BEGIN 3 */
  GPIOJ->ODR |= SEGDP_Pin;
  HAL_Delay(onTime);
  GPIOJ->ODR &= ~SEGDP_Pin;
  HAL_Delay(offTime);
}
/* USER CODE END 3 */
\end{lstlisting}

\subsection{b) Discussion:}
Wesentlich war hier die korrekte GPIO-Konfiguration in CubeMX. Ohne Umstellung auf Input Pull-Up konnten die Taster nicht sinnvoll eingelesen werden.

Die Lösung ist einfach und funktioniert zuverlässig mit festen Grenzen für die Zykluszeit. Eine Einschränkung ist, dass durch \texttt{HAL\_Delay()} die Reaktion auf Tastendrücke nur zwischen zwei Blinkphasen erfolgt.
